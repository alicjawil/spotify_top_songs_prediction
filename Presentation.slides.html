Predicting Hit Songs using Machine Learning¶

A song's popularity can be measured by its chart position.

For many musicians, getting their songs to chart high is of paramount importance in order garner a prominent presence in the music industry, and above all, to make money.

So now we ask:

  • What do all these hit songs have in common?
  • Is there a certain pattern or characteristics to determine a hit song?
  • What are the significant and influential predictors?

In this project, we attempt to classify a song as a hit or non-hit based on its audio features. We used various machine learning algorithms such as Logistic Regression, Gaussian Naive Bayes, Decision Trees, and Random Forests for this binary classification problem and discuss whether or not audio features provide enough information for determining a hit song.

Data Acquisition¶

We acquired data using Spotify's API for songs released in 2017. With Spotify's acquisition of Echo Nest, a music intelligence and data platform, their API allows users to access a song's audio features.

We pulled a total number of 7429 records, each categorized by 10 features.

Data Description¶

Features Description
acousticness A confidence measure from 0.0 to 1.0 of whether the track is acoustic. 1.0 represents high confidence the track is acoustic.
danceability Danceability describes how suitable a track is for dancing based on a combination of musical elements including tempo, rhythm stability, beat strength, and overall regularity. A value of 0.0 is least danceable and 1.0 is most danceable.
duration_ms The duration of the track in milliseconds.
energy Energy is a measure from 0.0 to 1.0 and represents a perceptual measure of intensity and activity.
instrumentalness The closer the instrumentalness value is to 1.0, the greater likelihood the track contains no vocal content. Values above 0.5 are intended to represent instrumental tracks, but confidence is higher as the value approaches 1.0.
liveness Detects the presence of an audience in the recording. Higher liveness values represent an increased probability that the track was performed live.
loudness The overall loudness of a track in decibels (dB). Values typical range between -60 and 0 db.
speechiness Speechiness detects the presence of spoken words in a track.
tempo The overall estimated tempo of a track in beats per minute (BPM).
valence A measure from 0.0 to 1.0 describing the musical positiveness conveyed by a track.
artist_popularity The popularity of the artist. The value will be between 0 and 100, with 100 being the most popular.

Data Analysis¶

Since this is a binary classification problem, we used the following the models:

  • Logistic Regression
  • Gaussian Naive Bayes
  • Decision Tree
  • Random Forests

Models were trained with a random 70% subset of our data, with the remaining 30% designated to be used as validation.

And for evaluating the model, we will need to take consideration that our model has class imbalance: 287 hits and 6358 no-hits.

We used confusion matrices to test the performance of our model. However, we cannot use Accuracy as an evaluation measure since our model can simply predict all as no-hits have achieve a ~95% accuracy rate.

Thus, we will use the AUC as it is a function of sensitivity and specificity, the curve is insensitive to disparities in the class proportions. Along with other metrics such as f1-score, precision and recall.

Dealing with Class Imbalance¶

Pre-processing will be done via means of:

  • Putting weights on errors proportional to the class imbalance.
    • Balanced vs Optimal Weighting
  • Re-sample the data to balance the positive/negatives classes.
    • SMOTE: Synthetic Minority Over-sampling Technique

Model 1: Logistic Regression¶

$ \begin{align} E(Y_i | X_i) = Pr(Y_i=1|X_i) = {\frac{1}{1 + e^{-(\beta_0 + \beta_1X_i)}}} \end{align} $

For any given x, The logistic response function will always yield to values in the interval [0,1].

Thus, if $(E(Y_i | X_i)<0.5)$, then the logistic model predicts $y = 0$ and, alternatively, if $E(Y_i | X_i)>0.5$ the model predicts $y = 1$.

Type of questions we attempted to answer with a binary logistic regression:

  • What predictors have an significant influence on the probability of a song being a hit?
  • How does the probability of a hit song change for every additional unit change in our predictors?

Model 2: Gaussian Naive Bayes¶

The second method that we will explore is Gaussian Naive Bayes.

As we learned in class, Naive Bayes incorporate the concept of conditional probability, or the probability of event A given that event B has occurred.

However, since we're dealing with continuous data, we'll have to make an assumption that the continuous values associated with each class are distributed according to a Gaussian distribution.

Hence, we will be looking at the probability of event (hit = 1) given that its predictor values are $x_i$ where the probability of $x_i$ can be computed by plugging $x_i$ into a Gaussian distribution parameterized by $\mu _{k}$ and $\sigma _{k}^{2}$.

Unfortunately, we weren't able to train a well-performing Gaussian Naive Bayes model and were only able to marginally beat the un-preprocessed model's performance.

Model 3: Decision (Classification) Trees¶

Decision trees are one of the most transparent and easy to interpret machine learning algorithms.

It involves separating observations into subgroups by creating splits on predictors.

And as such, these splits create logical rules that are easily understandable. An example logical rule could be, for example:

If artist popularity is >= X, then the probability of a song being a hit is Y

Gini Importance¶

Before we attempt pre-processing, we can attempt to further improve our model by actually selecting features that provides good impurity reduction (the measure based on which the (locally) optimal condition is chosen is called impurity).

The Gini impurity index is defined as:

$G=\sum {i=1}^n_c p_i(1-p_i)$

Where $n_c$ is the number of classes in the target variable and $p_i$ is the ratio of this class.

Model 4: Random Forests¶

To address the shortcomings of a single tree (like we trained above), with poor predictive power, we can attempt to combine results from multiple trees. This is an example of an ensemble method known as random forests (multiple trees = forests).

Random forests is basically conducted in a few steps:

  1. Draw multiple samples, with replacement, from our data (known as bootstrapping).
  2. Train a classification decision tree to each sample.
  3. Combine the classifications from the individual trees to obtain a majority voted result. (i.e. if we trained 100 trees, we classify a song as a hit if > 50 trees voted yes).
